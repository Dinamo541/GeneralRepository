1 ¬øC√≥mo funciona Java?
C++ compila a c√≥digo m√°quina. El ejecutable corre directamente en el hardware. Un .exe compilado en Windows no corre en Linux.

Java es diferente: el compilador (javac) convierte el c√≥digo a bytecode (.class), un formato intermedio. Ese bytecode lo ejecuta la JVM (Java Virtual Machine), instalada en cada sistema operativo. Resultado: "write once, run anywhere".
javac MiClase.java ‚Üí MiClase.class (bytecode) ‚Üí JVM ‚Üí ejecuta en cualquier OS
Memoria: en C++ la memoria se maneja con new/delete. En Java, la JVM tiene un Garbage Collector que limpia los objetos que ya no se usan. No hay punteros crudos, no hay delete, casi no hay memory leaks accidentales.

Todo es un objeto (casi). No hay funciones sueltas como en C++ ‚Äî todo vive dentro de una clase. El punto de entrada es siempre: 
public static void main(String[] args) { ... }
Tipos primitivos son la excepci√≥n: int, double, boolean, char, long, float no son objetos (por eficiencia). Sus versiones objeto son Integer, Double, Boolean, etc.
üí°
Packages = namespaces: en C++ se usa namespace. En Java se declara package com.miapp.logica; al inicio del archivo. Los imports reemplazan los #include, pero no copian c√≥digo ‚Äî solo indican al compilador d√≥nde buscar.
